路由注册

先看一个路由使用的基本栗子（来自vue-router官方examples）

vue-router作为vue的一个插件，在使用之前需要先使用Vue.use(VueRouter) 进行注册。
Vue 提供的插件注册机制是这样的，每个插件都需要自己实现一个静态的 install 方法，当我们执行 Vue.use 注册插件的时候，就会执行这个 install 方法，并且在这个 install 方法的第一个参数我们可以拿到 Vue 对象。
下面我们看看vue-router 的 install方法

函数入口先判断 vue-router 是否已经注册过，防止多次注册。接着干的一件重要的事情就是用 Vue.mixin 把 beforeCreate 和 destroyed 钩子函数注入到每一个组件中。beforeCreate里头先判断该组件是否是根实例，根实例的_routerRout指向自己，非根实例通过this._routerRoot = (this.$parent && this.$parent._routerRoot) || this_把每一个实例的_routerRoot都指向根实例。此外，根组件执行beforeCreate时，执行vue-router实例的init方法，这个之后介绍。并且用 defineReactive 方法把 this._route 变成响应式对象，用于在route发生变更是触发router-view组件的render函数，重新渲染组件，触发页面刷新。
对于 beforeCreate 和 destroyed 钩子函数，它们都会执行 registerInstance 方法，这个方法的作用之后会介绍。
之后将Vue.prototype.$router和Vue.prototype.$route指向this._routerRoot._router和this._routerRoot._route，vue实例就可以直接访问到vueRouter实例和当前的route。

最后注册两个全局组件，之后会介绍。

再看看match方法：

match 方法接收 3 个参数，其中 raw 是 RawLocation 类型，它可以是一个 url 字符串，也可以是一个 Location 对象；currentRoute 是 Route 类型，它表示当前的路径；redirectedFrom 和重定向相关，这里先忽略。match 方法返回的是一个路径，它的作用是根据传入的 raw 和当前的路径 currentRoute 计算出一个新的路径并返回，具体的细节请自行查阅源码。

Location: Location 的数据结构和浏览器提供的 window.location 部分结构有点类似，它们都是对 url 的结构化描述

Route: Route 表示的是路由中的一条线路，它除了描述了类似 Loctaion 的 path、query、hash 这些概念，还有 matched 表示匹配到的所有的 RouteRecord

这里url变化有两种情况：
